## 発表時間目安: 10-15分

## 1. イントロダクション (1分)

### スライド1: タイトル

**原稿:**
「本日は、Kaggleコンペティション『Recruit Restaurant Visitor Forecasting』を題材に、機械学習モデルの比較検証を行った結果を発表します。

このコンペでは、日本のレストランの来客数を予測します。データは、ホットペッパーグルメとAirレジという2つのシステムから提供されています。」

## 2. 問題設定 (2分)

### スライド2: なぜ来客予測が重要か

**原稿:**
「レストランにとって、来客数の予測は非常に重要です。

- **食材の仕入れ**: 多すぎれば廃棄、少なすぎれば品切れ
- **スタッフ配置**: 適切な人員配置でコスト削減
- **顧客満足度**: 待ち時間の短縮

しかし、予測は簡単ではありません。天気、祝日、競合店など、多くの要因が影響します。」

### スライド3: 評価指標 RMSLE

**原稿:**
「評価指標はRMSLE、Root Mean Squared Logarithmic Errorです。

この指標の特徴は、**過少予測をより厳しく罰する**ことです。

なぜでしょうか？

レストランにとって、食材が足りなくなる（過少予測）方が、余る（過大予測）より深刻だからです。余った食材は翌日使えますが、足りないとお客様に迷惑がかかります。」

## 3. データの事前処理 (3分)

### スライド4: データ概要

**原稿:**
「データは8つのファイルから構成されています。

主要なファイルは：
- `air_visit_data.csv`: 来客履歴（約25万件）
- `air_store_info.csv`: 店舗情報（ジャンル、エリア）
- `date_info.csv`: 祝日情報

期間は2016年1月から2017年4月までの約16ヶ月分です。」

### スライド5: データの問題点と対処

**原稿:**
「データにはいくつかの問題がありました。

**問題1: 欠損日**
営業していない日はデータがありません。
→ 対処: 日付でリサンプルし、欠損日は0で補完

**問題2: 外れ値**
一部の店舗で異常に多い来客数
→ 対処: 2.4σ（標準偏差）を超える値をクリップ

**問題3: 新規店舗**
履歴が少ない店舗は予測が困難
→ 対処: 同じジャンル・エリアの平均値で補完」

### スライド6: 特徴量エンジニアリング

**原稿:**
「上位解法を分析した結果、**特徴量エンジニアリングが最も重要**だとわかりました。

作成した特徴量：

1. **時間特徴量**: 曜日、月、祝日フラグ
2. **Rolling統計量**: 過去7日、14日、21日の平均・標準偏差
3. **ラグ特徴量**: 1週間前、2週間前の来客数
4. **指数加重平均**: 直近のデータを重視した平均

特にRolling統計量が最も予測に寄与しました。」

## 9. 改善点とその理由 - なぜ精度が上がったか (3分)

### スライド17: 精度改善の全体像

**原稿:**
「ここで、なぜモデルを変えると精度が上がったのか、その理由を詳しく解説します。

改善は大きく2つの観点から説明できます：
1. **アルゴリズムの違い**（バギング vs ブースティング）
2. **実装の最適化**（XGBoost vs LightGBM）」

### スライド18: バギング vs ブースティング

**原稿:**
「まず、ランダムフォレストと勾配ブースティングの根本的な違いです。

**ランダムフォレスト（バギング）:**
```
木1 → 予測A
木2 → 予測B    → 平均して最終予測
木3 → 予測C
（各木は独立して学習）
```

**勾配ブースティング（XGBoost/LightGBM）:**
```
木1 → 誤差を計算
     ↓
木2 → 木1の誤差を修正    → 累積して最終予測
     ↓
木3 → 木2までの誤差を修正
（各木が前の誤差を学習）
```

**なぜブースティングが優れるか:**
- 誤差を段階的に修正するため、より細かいパターンを学習可能
- 難しいサンプルに集中的に対応できる
- 結果として、同じ木の数でも高い精度を達成」

### スライド19: ランダムフォレストの限界

**原稿:**
「ランダムフォレストが精度で劣る具体的な理由です。

**理由1: 平均化による情報の損失**
各木の予測を平均するため、極端な予測が薄まります。
来客数が急増・急減するケースを捉えにくい。

**理由2: 誤差の蓄積**
一度間違えたパターンを修正する仕組みがありません。
同じミスを繰り返す可能性があります。

**理由3: 相関のある木**
ブートストラップサンプリングを使いますが、
重要な特徴量が同じなら、似たような木ができてしまいます。」

### スライド20: XGBoostの改善ポイント

**原稿:**
「XGBoostがランダムフォレストより優れる理由です。

**改善点1: 勾配ベースの学習**
前の木の予測誤差（勾配）を次の木が学習します。
→ 誤差を効率的に減らせる

**改善点2: 正則化**
L1/L2正則化により過学習を防止。
```
目的関数 = 損失 + λ × 複雑さペナルティ
```
→ 汎化性能が向上

**改善点3: 欠損値の自動処理**
欠損値を持つデータも、最適な分岐方向を自動で決定。
→ 前処理の手間が減り、情報を失わない

**結果: RMSLE 0.54 → 0.52（約4%改善）**」

### スライド21: LightGBMの改善ポイント

**原稿:**
「LightGBMがXGBoostより優れる理由です。

**改善点1: Leaf-wise成長戦略**
```
XGBoost: Level-wise（層ごとに均等に成長）
    [    ]
   /      \
 [ ]      [ ]

LightGBM: Leaf-wise（損失が大きい葉を優先成長）
    [    ]
   /      \
 [詳細]   [ ]
  / \
[  ] [  ]
```
→ 同じ計算量でより深い学習が可能

**改善点2: ヒストグラムベースのアルゴリズム**
連続値を離散化（ビン化）して計算を高速化。
```
元データ: [1.2, 3.5, 2.1, 4.8, ...]
ビン化:   [1,   3,   2,   4,   ...]
```
→ 計算量: O(データ数) → O(ビン数)
→ 10倍以上高速化、しかも精度はほぼ同等

**改善点3: カテゴリ変数の直接処理**
One-hotエンコーディング不要で、最適な分割を自動発見。
→ メモリ効率が向上、特徴量の爆発を防止

**結果: RMSLE 0.52 → 0.50（約4%改善）**」

### スライド22: 精度改善のまとめ

**原稿:**
「精度改善の流れをまとめます。

| 改善ステップ | RMSLE | 改善幅 | 主な理由 |
|-------------|-------|--------|----------|
| Random Forest | 0.54 | - | ベースライン |
| → XGBoost | 0.52 | -0.02 | 勾配ブースティング採用 |
| → LightGBM | 0.50 | -0.02 | Leaf-wise + ヒストグラム |

**総改善: 0.54 → 0.50（約7.4%改善）**

重要なポイント：
1. アルゴリズムの選択で大きな差が出る
2. 同じ勾配ブースティングでも実装で差が出る
3. 速度向上は精度向上にもつながる（より多くの実験が可能）」

## 補足: Q&A想定

**Q: なぜランダムフォレストは精度が劣るのか？**
A: ランダムフォレストは各木が独立して学習するバギング手法です。一方、勾配ブースティングは前の木の誤差を修正しながら学習するため、より精度が高くなります。

**Q: LightGBMが速い理由は？**
A: Histogram-based アルゴリズムとLeaf-wise成長戦略を採用しているためです。データを離散化することで計算量を削減しています。

**Q: 実務でもLightGBMを使うべきか？**
A: テーブルデータの回帰・分類タスクでは、LightGBMは非常に有力な選択肢です。ただし、解釈性が重要な場合は線形モデルや決定木も検討すべきです。
